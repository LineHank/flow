# Dify流程图仿制

# 画布右键菜单配置说明 

本文档说明 `flow-config.js` 中 `contextMenu.container.menulists`（第 190-256 行）的作用，即画布空白处右键时弹出的菜单项及其功能。

## 菜单结构概览

```
流程图信息
添加节点
  ├── 基础
  │   ├── 开始
  │   ├── 串行节点
  │   ├── 并行节点
  │   └── 结束
  ├── 高级
  │   ├── 虚拟节点
  │   ├── 节点任务
  │   └── 子流程
  └── 泳道
      ├── 横向泳道
      └── 纵向泳道
对齐方式
  ├── 垂直左对齐
  ├── 垂直居中
  ├── 垂直右对齐
  ├── 水平上对齐
  ├── 水平居中
  └── 水平下对齐
全选
粘贴
发布流程
```

---

## 一、基础菜单项

### 1. 流程图信息 (flowInfo)

显示当前流程图的基本统计信息：节点数量、连线数量。用于快速了解流程图规模。

---

### 2. 添加节点 (children)

在右键点击位置添加各类流程节点。节点按类别分为：**基础**、**高级**、**泳道**。

#### 2.1 基础节点

| 节点类型 | fnHandler | 说明 |
|---------|-----------|------|
| 开始 | addNodeStart | 流程起始节点，每个流程有且仅有一个 |
| 串行节点 | addNodeSerial | 串行审批节点，见下文详解 |
| 并行节点 | addNodeParallel | 并行审批节点，见下文详解 |
| 结束 | addNodeEnd | 流程终止节点 |

**串行节点 vs 并行节点的区别：**

| 对比项 | 串行节点 (SERIAL) | 并行节点 (PARALLEL) |
|--------|-------------------|---------------------|
| 执行顺序 | 顺序执行，一个任务完成后才会进入下一个 | 同时执行，多个分支可并行流转 |
| 任务数量 | 通常对应 1 个任务 | 可配置多个任务（jobSize） |
| 典型场景 | 单人审批、单步操作 | 会签、多部门同时审批 |
| 流转逻辑 | 当前任务完成 → 下一任务开始 | 所有分支任务完成后 → 汇聚到下一节点 |

**示例：**

- **串行**：申请人 → 部门经理审批 → 总经理审批（依次进行）
- **并行**：申请人 → 财务+法务+技术同时审批 → 汇总（三个部门同时处理）

#### 2.2 高级节点

| 节点类型 | fnHandler | 说明 |
|---------|-----------|------|
| 虚拟节点 | addNodeVirtual | 不产生实际任务，用于流程结构控制（如条件分支汇聚） |
| 节点任务 | addNodeJob | 挂载在串行/并行节点上的具体审批任务，可配置办理人、角色等 |
| 子流程 | addNodeChildFlow | 调用另一流程作为子流程，实现流程嵌套 |

#### 2.3 泳道

| 节点类型 | fnHandler | 说明 |
|---------|-----------|------|
| 横向泳道 | addNodeXLane | 水平方向的泳道容器，用于按角色/部门划分流程区域 |
| 纵向泳道 | addNodeYLane | 垂直方向的泳道容器 |

泳道用于在流程图中按职能或组织划分区域，使流程结构更清晰。

---

### 3. 对齐方式 (children)

对已选中的多个节点进行排列对齐。需先多选节点（Ctrl + 鼠标框选）后再使用。

| 菜单项 | fnHandler | 说明 |
|--------|-----------|------|
| 垂直左对齐 | verticaLeft | 以第一个节点为基准，纵向排列，左边缘对齐 |
| 垂直居中 | verticalCenter | 纵向排列，垂直居中 |
| 垂直右对齐 | verticalRight | 纵向排列，右边缘对齐 |
| 水平上对齐 | levelUp | 以第一个节点为基准，横向排列，上边缘对齐 |
| 水平居中 | levelCenter | 横向排列，水平居中 |
| 水平下对齐 | levelDown | 横向排列，下边缘对齐 |

---

### 4. 全选 (selectAll)

选中画布上所有节点，便于批量移动或删除。

---

### 5. 粘贴 (paste)

粘贴之前通过节点右键菜单「复制节点」复制的内容。新节点会生成新 ID，位置在鼠标所在处略有偏移。

---

### 6. 发布流程 (publishFlow)

将当前流程图发布保存到后端。

---

## 二、配置字段说明

每个菜单项支持以下字段：

| 字段 | 说明 |
|------|------|
| btnName | 菜单显示名称 |
| fnHandler | 点击后调用的方法名，需在 `flow-area.vue` 中实现 |
| icoName | 菜单项左侧图标样式（vue-contextmenu 使用） |
| children | 子菜单列表，存在则显示为带箭头的子菜单 |
| disabled | 是否禁用 |

---

## 三、相关文件

- **配置定义**：`src/config/flow-config.js`
- **事件处理**：`src/components/flow-area.vue`（`@addNodeStart`、`@verticaLeft` 等）
- **节点模板**：`src/config/node-config.js`（各节点默认属性）
